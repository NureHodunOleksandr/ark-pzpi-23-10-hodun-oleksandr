Мета роботи
     Метою лабораторної роботи є розгортання серверної частини програмної системи у хмарному середовищі, налаштування підключення до хмарної бази даних, перевірка коректності роботи розгорнутої системи та демонстрація її функціональності.
     Хід виконання роботи
     На початку я підготував серверну частину програної системи, реалізовану з використанням середовища виконання Node.js та фреймворку Express.js. Код серверної частини було структуровано відповідно до архітектури клієнт-серверної системи та розміщено у репозиторії GitHub “https://github.com/NureHodunOleksandr/planner-server”.
     У репозиторії наявні необхідні файли для розгортання:
     * package.json – для керування залежностями;
     * server.js – точка входу серверного застосунку;
     * директорія prisma/ з описом схеми бази даних та конфігурацією ORM Prisma.
     1. Створення та налаштування хмарної бази даних
     Для зберігання даних програмної системи було використано хмарний сервіс Aiven, у якому створено базу даних MySQL.
     Під час налаштування бази даних було:
     * створено MySQL-сервіс;
     * отримано параметри підключення (host, port, ім’я бази даних, користувач, пароль);
     * сформовано рядок підключення DATABASE_URL;
     * налаштовано мережевий доступ (IP allowlist) для забезпечення підключення серверної частини з хмарного середовища Render;
     * увімкнено захищене SSL-зʼєднання.

     2. Розгортання серверної частини у хмарному середовищі
     Для розгортання серверної частини програмної системи було використано хмарну платформу Render.
     Під час розгортання виконано такі дії:
     * створено новий Web Service;
     * підключено GitHub-репозиторій із серверним кодом;
     * обрано середовище виконання Node.js;
     * налаштовано команду збірки для встановлення залежностей та генерації Prisma Client;
     * налаштовано змінні середовища, зокрема DATABASE_URL, для підключення до хмарної бази даних.

     3. Запуск та перевірка роботи серверної частини
     Після успішного розгортання серверної частини програмної системи для перевірки я виконав запуск застосунку. У логах хмарного сервісу Render підтверджено успішний запуск сервера та його готовність до обробки HTTP-запитів.
     Для перевірки працездатності системи зробимо:
     * відкриємо публічний URL серверної частини у браузері;
     * отримаємо відповідь сервера, що підтверджує коректну роботу API;
     * виконанємо тестові HTTP-запити за допомогою інструментів Thunder Client;
     * підтвердемо коректну роботу маршрутизації та взаємодії з базою даних.
